# REQUIREMENTS.md

项目名称：Silence（肃静）
组织：Lapis Lazuli
域名（建议）：silence.lapis-lazuli.org

概述：
Silence 是一个完全私密、管理员（Madam Lazulis）完全掌控的社区平台。核心特点包括：问卷+人工审批进入、强制邮箱绑定、完全私密内容存储、定制货币（Ludus chartarum，简称 LC）、商品上架/购买须管理员审批、个人化关系导图（私有）、交互型论坛（特定术语）、私信与数据分析。所有操作与文本均使用平台自有术语（例如：认同数、记录数、交互数、主体存在等）。

必须实现（2 周内核心优先级）：
1. 问卷系统（30题，前 10 题已提交）——答题通过才能申请注册；未通过者被拒绝并被重定向。
2. 注册流程：邮箱绑定 + 验证邮件 + 提交问卷 → 管理员审批后激活。
3. 管理员 Dashboard：审批注册、审批充值、审批上架、用户管理、数据库导出。
4. LC 货币系统：用户发起充值申请（法币换 LC），管理员人工审批到账。
5. 社区模块：帖子发布（文本/图片/链接）、分类（交流部/理论部等）、交互数/认同数/记录数计数。
6. 商品模块：发布商品须管理员审批，购买使用 LC。
7. 私信：服务器端加密存储；端对端为可选（初版不做）。
8. 关系导图：用户私有定义关系并可视化，交互数据仅作参考。
9. 完全私密：站点不被搜索引擎索引、邀请制、内容由管理员最终审核。
10. 管理密钥：只有持有管理员密钥者可在代码层面进行改动。

安全与私密：
- 全站使用 HTTPS。数据库字段敏感信息加密存储。管理员在 Dashboard 可以查看用户真实身份（审批所需）。
- 匿名发帖选项：显示为“主体存在（随机编号）”，管理员在必要时有权限查证。
- 所有内容仅在你授权下对内可见；不同节点（多域名）数据相互隔离，无跨节点传播。

运维与扩展：
- 支持将来接入多个节点（每个节点为独立站点、独立数据库），中心化组织管理策略。
- 性能：支持水平扩展（后期可接入 CDN、负载均衡、读写分离）。
